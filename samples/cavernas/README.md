This is the example used to develop the **Hot-Reload** and **Fluid Movement** infrastructure of Stargateldtk.

## Contents
- `data/`: The `world.ldtk` file defining the map.
- `assets/`: Necessary cavern and player sprites.
- `main.rb`: An example of how to orchestrate everything.

## What You Will Learn Here
1. **Layer Rendering**: How to iterate over `world.grids` to draw sprites.
2. **Cinematic Camera**: How to follow the player with smoothing.
3. **Hot-Reload**: The pattern for detecting changes in LDtk and reloading the map instantly.
4. **Interpolation**: How to separate logical and visual positions for fluid movement.
